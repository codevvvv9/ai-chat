# API 文档

本文档用于前后端协作，记录当前后端已经实际暴露的接口。这里的接口仍然是最小占位实现，主要用于联调、连通性验证和后续扩展，不代表最终业务 API 设计。

## 基础说明

- 后端基于 `Express + TypeScript`
- 当前只注册了两个显式路由：`GET /health` 和 `GET /api/example`
- 请求与响应均使用 JSON
- 目前没有认证、分页、查询参数、请求体校验等业务能力
- 除已注册路由外，其余路径都会返回统一的 `404` JSON 响应

## 接口列表

### 1. 健康检查

- 请求方法：`GET`
- 路径：`/health`
- 说明：用于确认服务是否存活

响应示例：

```json
{
  "status": "ok"
}
```

### 2. 示例接口

- 请求方法：`GET`
- 路径：`/api/example`
- 说明：占位示例接口，用于验证路由和返回值链路

响应示例：

```json
{
  "message": "example placeholder"
}
```

### 3. 未命中路由

- 请求方法：任意
- 路径：未被上述路由匹配的任意地址
- 说明：统一返回 `404`

响应示例：

```json
{
  "message": "Not found"
}
```

## 当前状态

这份 API 目前是占位性质，后端只提供最小可运行骨架。后续如果新增业务接口，需要同步补充：

- 请求方法
- 路径
- 请求参数或请求体
- 响应结构
- 错误码和错误响应

## 约定

- 在当前阶段，不要假设存在未列出的接口
- 前后端联调时，以这里和后端 `src/routes/` 下的实际代码为准
- 若接口行为变更，应优先更新后端代码，再同步更新本文档
